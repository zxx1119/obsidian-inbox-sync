# obsidian-inbox-sync 改造说明

## v0.3.3 — 文件名加日期前缀，按时间排序

### 问题

Obsidian 文件树默认按文件名排序（字母/拼音序），笔记标题又是正文前几字，
导致笔记在文件树里的顺序跟创建时间无关，看起来很乱。

### 改动

文件名前面加笔记创建日期前缀 `YY-MM-DD`：

```
之前：这两天看到一个说法，说在国内其实没有真正.md
现在：25-06-15 这两天看到一个说法，说在国内其实没有真正.md
```

- 用 `createdAt`（笔记初始创建时间，固定不变），不用 `updatedAt`
- 2 位年份（如 25），更短更直观
- frontmatter 的 `title` 字段保持原样（不含日期），方便搜索原标题
- 同一天多条笔记仍按标题拼音排（精确到日期，不精确到分钟）

### 影响面

- **新同步的笔记** → 自动带日期前缀
- **同一条笔记更新时** → 自动迁移到带日期前缀的名字（路径变化检测会处理）
- **现有笔记** → 不动（旧笔记迁移由用户自行处理）
- **批注嵌入引用 `[[父笔记文件名]]`** → 自动用新文件名
- **`[[note-xxx]]` 链接转换** → 自动用新文件名

## v0.3.2 — 代码审查修复

系统审查后修复 13 个问题，按严重程度分 P0/P1/P2/P3 四级。

### P0 严重问题（死代码）

1. **删除死代码 `webdav-client.ts`**
   - 原文件里的 `WebDAVClient` 类从未被引用，实际用的是 `webdav-native.ts` 的 `WebDAVNativeClient`
   - 两者 `listNotes()` 路径不一致（一个查 `batch-backup/notes/`，一个查 `notes/`），保留死代码会让维护者混淆

2. **删除 `withRetry` 死代码**
   - `sync-manager.ts` 的 `withRetry` 方法定义了但从未被调用
   - 下载笔记/资源没有重试逻辑，单次失败就失败。未来若要加重试再实现

### P1 逻辑漏洞

3. **内联模式切换时清理旧文件**
   - 问题：`inlineAnnotations` 从 false→true 时，旧的独立批注 .md 文件不会被清理，会跟新的内联批注共存
   - 修复：检测到该配置变化时，递归扫描删除所有带 `parent_id` frontmatter 的 .md 文件
   - `checkSettingsChanged` 改为返回具体变化项，主流程根据变化项决定清理动作

4. **移除未实现的假配置**
   - `preserveContentTags`：UI 有开关但代码永远保留标签，开关无效
   - `conflictResolution`：UI 有下拉但代码永远是 overwrite，skip/rename 策略未实现
   - 修复：从设置页移除这两个选项（settings 字段保留兼容旧 data.json，标记 @deprecated）

### P2 防御性修复

5. **`cloudClient` undefined 防御**
   - `initializeClients` 如果 storageType 异常（data.json 被改坏），cloudClient 会是 undefined
   - 修复：storageType 不匹配时抛明确错误，避免 undefined 传给 AssetHandler 埋雷

6. **修正标题预览清洗破坏中文标点**
   - 问题：`/[#*>_`~|-]/g` 会误删中文破折号 `—` 和省略号
   - 修复：只清洗 ASCII 范围的 markdown 符号，行首符号单独处理，行内只删 `_` `` ` `` `*`

### P3 代码质量

7. **合并重复常量**：`ANNOTATION_BLOCK_START` 和 `ANNOTATION_INLINE_START` 值相同，合并为一个
8. **修正 `note-parser.ts` 过时注释**：注释还在说 v0.3.0 的"笔记名-assets"，已更新为 v0.3.1 的全局 `inBox/assets/`
9. **简化 `findNoteParentId` 返回类型**：从 `{ noteId, parentId }` 简化为 `string | null`（调用方只用 parentId）
10. **脱敏 debug 日志**：WebDAV 不再打印用户名和 Base64 认证信息，S3 不再打印 AccessKey 前 4 位
11. **修正 `listNotes` 跳过逻辑**：原代码 `for (let i = 1; ...)` 靠顺序跳过目录本身，改为按文件名 `.json` 后缀过滤，更健壮

## v0.3.1 — 修复笔记点不开、图片挤掉正文的问题

### 问题

v0.3.0 把图片存到 `笔记名-assets/` 子文件夹后出现两个毛病：

1. **笔记在 Obsidian 文件树里点不开**——因为笔记名本身是一句话（比如"这两天看到一个说法，说在国内其实没有真正"），生成的 `笔记名-assets/` 文件夹跟 `.md` 文件名几乎一样，Obsidian 把它们折叠成一个节点，点进去只看到图片，找不到正文入口。
2. **正文被图片挤掉**——图片用 `![]()` 嵌入式引用，打开笔记图片直接渲染，正文只剩一行被压在中间。

### 改动

1. **图片统一存到 vault 根的 `inBox/assets/` 目录**
   - 不再每篇笔记建 `笔记名-assets/` 子文件夹
   - `.md` 文件不再被 Obsidian 折叠成"文件夹节点"，点开就是正文
   - 跨设备同步（同 vault 结构）路径稳定，不会因为盘符变化失效

2. **图片引用从 `![]()` 改成 `[]()`**
   - 图片不再直接渲染进笔记，正文不会被挤掉
   - 点击链接即可在 Obsidian 里跳转查看原图
   - 视频、音频也一并改成跳转链接

3. **批注资源同步走全局 `inBox/assets/`**
   - 跟主笔记保持一致，不再用父笔记名建子文件夹

### 不影响的部分

- 现有已同步的笔记**不动**（旧 `笔记名-assets/` 文件夹保持原样，需要的话用户可自行迁移或删除）
- 笔记正文、批注、标签逻辑全部不变
- 只影响新同步的笔记

### 涉及文件

- `src/sync/asset-handler.ts` — `getAssetLocalPath` 改返回 `inBox/assets/文件名`
- `src/sync/markdown-writer.ts` — `buildAssetBlock` / `buildAnnotationBlock` / `collectAssetLines` 改 assetDir 和引用格式

## v0.3.0 — 图片跟笔记走 + 正文显示图片

针对原插件的两个不足做了改造：

### 问题一：批注单独成文件，与原文割裂

**原状**：每条批注是独立的 `.md` 文件，平铺在 `inBox/` 目录。父笔记末尾用 `![[批注文件名]]` 嵌入引用，但批注内容和原文没法对照着看。

**改造**：新增 `inlineAnnotations` 配置项（默认开启）。开启后：
- 批注不再生成独立文件
- 批注内容直接拼到父笔记末尾的「## 批注」区块
- 每条批注带标题、时间戳、正文
- 父笔记成为一份完整的、可对照阅读的文档

**效果示例**：
```markdown
---
title: 今日记录
inbox_id: note-abc123
tags:
  - 日记/生活
---

今天天气不错 #心情/开心

---

## 批注

### 补充
> _2026-04-11 14:30_

后来下雨了，带了伞。

### 回顾
> _2026-04-12 09:15_

那天其实没下雨，白带了。
```

关闭 `inlineAnnotations` 则回到旧的「独立文件 + 嵌入引用」模式。

## 问题二：笔记平铺，没有按标签聚合

**原状**：所有笔记平铺在 `inBox/` 一个目录里，标签只写进 frontmatter，没有任何目录组织。

**改造**：新增 `organizeByTag` 配置项（默认开启）。开启后：
- 按主标签（笔记的第一个标签）建子文件夹归档
- 支持嵌套标签：`#日记/生活` → `inBox/日记/生活/note.md`
- 无标签笔记留在根目录 `inBox/note.md`
- 其余标签仍保留在 frontmatter 的 `tags` 字段里

**目录结构示例**：
```
inBox/
├── 日记/
│   └── 生活/
│       ├── 2026-04-10.md
│       └── 2026-04-11.md
├── 技术/
│   └── 前端/
│       └── React学习笔记.md
└── 无标题-20260412.md   ← 无标签笔记留在根目录
```

可选配置 `tagFolderRoot`：如果不想标签目录直接建在 `inBox/` 下，可以指定一个父目录名，比如设为 `tags`，则路径变成 `inBox/tags/日记/生活/note.md`。

## 新增配置项

在 Obsidian 设置 → inBox 同步设置 → 笔记组织方式 中可配置：

| 配置项 | 默认值 | 说明 |
|--------|--------|------|
| 按标签分文件夹 | 开启 | 按主标签建子文件夹归档笔记 |
| 标签目录根 | 空 | 标签子文件夹的父目录名，留空则直接在存储路径下建 |
| 批注内联到父笔记 | 开启 | 批注内容拼到父笔记末尾，不单独生成文件 |

## 改动的文件

- `src/types/settings.ts` — 新增 3 个配置字段
- `src/sync/markdown-writer.ts` — 重写，支持标签分目录 + 批注内联
- `src/sync/sync-manager.ts` — 调整同步流程为两阶段（先解析收集批注，再写入）
- `src/ui/settings-tab.ts` — 新增配置 UI

## 安装

构建后把 `main.js` 和 `manifest.json` 复制到你 vault 的插件目录：
```
<vault>/.obsidian/plugins/inbox-sync/
```

然后在 Obsidian 设置中重新启用插件即可。新配置项默认开启，无需额外操作。

## 注意

- 首次启用新逻辑后，旧的平铺笔记不会自动迁移，建议先清空 `inBox/` 目录再重新同步一次
- 如果想保留旧的扁平结构，关闭「按标签分文件夹」和「批注内联到父笔记」即可回到原行为

# v0.3.0 更新说明（2026-06-24）

本次更新解决两个问题：**图片跟笔记走 + 正文显示图片**、**无标题笔记用正文前几字当标题**。

## 更新一：图片跟着笔记走，正文直接显示图片

### 问题
之前图片统一下载到 `inBox/assets/images/` 全局目录，导致：
- 笔记在 `inBox/日记/生活/今天.md`，图片却跑去 `inBox/assets/images/`，图文分离
- 笔记正文里没有图片引用 `![](...)`，打开笔记只看到文字，图片在另一个文件夹躺着
- Obsidian 把 `assets/images` 目录路径误识别成标签 `assets/images` 显示

### 改造
- **图片跟着笔记走**：图片下载到笔记同目录下的 `笔记名-assets/` 子文件夹
  - 例如笔记 `inBox/日记/生活/今天.md`，图片存到 `inBox/日记/生活/今天-assets/img-001.jpg`
- **正文显示图片**：笔记正文末尾自动追加图片引用，用相对路径
  ```markdown
  今天去公园散步，天气很好

  ---

  ![](今天-assets/img-001.jpg)

  ![](今天-assets/img-002.jpg)
  ```
- **视频/音频/附件同理**：都放 `笔记名-assets/` 下，正文追加引用
- **内联批注的资源**：批注的图片放父笔记的 assets 文件夹，引用在批注块里
- **不再产生 `assets/images` 标签**：图片不再堆在全局目录，Obsidian 不会误识别

### 目录结构对比

**改之前**（图文分离）：
```
inBox/
├── 日记/
│   └── 生活/
│       └── 今天.md          ← 只有文字
└── assets/
    └── images/
        ├── img-001.jpg      ← 图片在这
        └── img-002.jpg
```

**改之后**（图文在一起）：
```
inBox/
└── 日记/
    └── 生活/
        ├── 今天.md           ← 正文里有 ![](今天-assets/img-001.jpg)
        └── 今天-assets/
            ├── img-001.jpg
            └── img-002.jpg
```

## 更新二：无标题笔记用正文前几个字当标题

### 问题
之前无标题笔记用日期当文件名，如 `2026-04-11 14.30.22.md`，看不出内容是啥。

### 改造
- **无标题笔记**：取正文前 20 个字当标题（文件名）
- **清洗规则**：去掉 `#标签`、markdown 语法符号（`#`、`*`、`>`、`-`、`![]()`、`[[]]` 等）、空白字符
- **有标题笔记**：完全不动，用原标题
- **纯图片/纯标签笔记**（清洗后正文为空）：回退到日期格式
- **日期不会丢**：日期保留在 frontmatter 的 `created` 字段里

### 效果示例

| 笔记内容 | 改之前的文件名 | 改之后的文件名 |
|---------|--------------|--------------|
| 有标题"今日记录" | `今日记录.md` | `今日记录.md`（不变） |
| 无标题，正文"今天去公园散步..." | `2026-04-11 14.30.22.md` | `今天去公园散步.md` |
| 无标题，正文"#日记 今天开心" | `2026-04-11 14.30.22.md` | `今天开心.md` |
| 无标题，纯图片无文字 | `2026-04-11 14.30.22.md` | `2026-04-11 14.30.22.md`（回退日期） |

## 改动的文件

- `src/sync/note-parser.ts` — `getLocalPath` 不再写死 `assets/images/`，localPath 改为只存文件名
- `src/sync/asset-handler.ts` — `handleAssets` 接收笔记 filePath 参数，资源下载到笔记同目录的 `笔记名-assets/` 子文件夹
- `src/sync/markdown-writer.ts` —
  - `getDisplayTitle`：无标题时取正文前 20 字（清洗后），空则回退日期
  - `generateMarkdown`：正文后追加资源引用区块（图片/视频/音频/附件）
  - `buildAnnotationBlock`：批注块里追加批注的资源引用
  - 新增 `extractContentPreview`、`buildAssetBlock`、`collectAssetLines`、`extractAssetFileName` 方法
- `src/sync/sync-manager.ts` — 调用 `handleAssets` 时传入笔记 filePath

## 升级建议

1. 编译生成新的 `main.js`
2. 把 `main.js` 和 `manifest.json` 复制到 vault 的插件目录
3. 建议先清空旧的 `inBox/assets/` 目录（里面是旧的全局图片），然后重新同步一次，图片会按新规则下载到各笔记旁边
4. 无标题笔记的文件名会在下次同步时自动更新为正文前几字

## 兼容性

- 有标题的笔记完全不受影响
- 旧的 `inBox/assets/` 目录里的图片不会被自动删除，需手动清理
- 配置项无变化，无需重新配置
