# obsidian-inbox-sync 改造说明

针对原插件的两个不足做了改造：

## 问题一：批注单独成文件，与原文割裂

**原状**：每条批注是独立的 `.md` 文件，平铺在 `inBox/` 目录。父笔记末尾用 `![[批注文件名]]` 嵌入引用，但批注内容和原文没法对照着看。

**改造**：新增 `inlineAnnotations` 配置项（默认开启）。开启后：
- 批注不再生成独立文件
- 批注内容直接拼到父笔记末尾的「## 批注」区块
- 每条批注带标题、时间戳、正文
- 父笔记成为一份完整的、可对照阅读的文档

**效果示例**：
```markdown
---
title: 今日记录
inbox_id: note-abc123
tags:
  - 日记/生活
---

今天天气不错 #心情/开心

---

## 批注

### 补充
> _2026-04-11 14:30_

后来下雨了，带了伞。

### 回顾
> _2026-04-12 09:15_

那天其实没下雨，白带了。
```

关闭 `inlineAnnotations` 则回到旧的「独立文件 + 嵌入引用」模式。

## 问题二：笔记平铺，没有按标签聚合

**原状**：所有笔记平铺在 `inBox/` 一个目录里，标签只写进 frontmatter，没有任何目录组织。

**改造**：新增 `organizeByTag` 配置项（默认开启）。开启后：
- 按主标签（笔记的第一个标签）建子文件夹归档
- 支持嵌套标签：`#日记/生活` → `inBox/日记/生活/note.md`
- 无标签笔记留在根目录 `inBox/note.md`
- 其余标签仍保留在 frontmatter 的 `tags` 字段里

**目录结构示例**：
```
inBox/
├── 日记/
│   └── 生活/
│       ├── 2026-04-10.md
│       └── 2026-04-11.md
├── 技术/
│   └── 前端/
│       └── React学习笔记.md
└── 无标题-20260412.md   ← 无标签笔记留在根目录
```

可选配置 `tagFolderRoot`：如果不想标签目录直接建在 `inBox/` 下，可以指定一个父目录名，比如设为 `tags`，则路径变成 `inBox/tags/日记/生活/note.md`。

## 新增配置项

在 Obsidian 设置 → inBox 同步设置 → 笔记组织方式 中可配置：

| 配置项 | 默认值 | 说明 |
|--------|--------|------|
| 按标签分文件夹 | 开启 | 按主标签建子文件夹归档笔记 |
| 标签目录根 | 空 | 标签子文件夹的父目录名，留空则直接在存储路径下建 |
| 批注内联到父笔记 | 开启 | 批注内容拼到父笔记末尾，不单独生成文件 |

## 改动的文件

- `src/types/settings.ts` — 新增 3 个配置字段
- `src/sync/markdown-writer.ts` — 重写，支持标签分目录 + 批注内联
- `src/sync/sync-manager.ts` — 调整同步流程为两阶段（先解析收集批注，再写入）
- `src/ui/settings-tab.ts` — 新增配置 UI

## 安装

构建后把 `main.js` 和 `manifest.json` 复制到你 vault 的插件目录：
```
<vault>/.obsidian/plugins/inbox-sync/
```

然后在 Obsidian 设置中重新启用插件即可。新配置项默认开启，无需额外操作。

## 注意

- 首次启用新逻辑后，旧的平铺笔记不会自动迁移，建议先清空 `inBox/` 目录再重新同步一次
- 如果想保留旧的扁平结构，关闭「按标签分文件夹」和「批注内联到父笔记」即可回到原行为

# v0.3.0 更新说明（2026-06-24）

本次更新解决两个问题：**图片跟笔记走 + 正文显示图片**、**无标题笔记用正文前几字当标题**。

## 更新一：图片跟着笔记走，正文直接显示图片

### 问题
之前图片统一下载到 `inBox/assets/images/` 全局目录，导致：
- 笔记在 `inBox/日记/生活/今天.md`，图片却跑去 `inBox/assets/images/`，图文分离
- 笔记正文里没有图片引用 `![](...)`，打开笔记只看到文字，图片在另一个文件夹躺着
- Obsidian 把 `assets/images` 目录路径误识别成标签 `assets/images` 显示

### 改造
- **图片跟着笔记走**：图片下载到笔记同目录下的 `笔记名-assets/` 子文件夹
  - 例如笔记 `inBox/日记/生活/今天.md`，图片存到 `inBox/日记/生活/今天-assets/img-001.jpg`
- **正文显示图片**：笔记正文末尾自动追加图片引用，用相对路径
  ```markdown
  今天去公园散步，天气很好

  ---

  ![](今天-assets/img-001.jpg)

  ![](今天-assets/img-002.jpg)
  ```
- **视频/音频/附件同理**：都放 `笔记名-assets/` 下，正文追加引用
- **内联批注的资源**：批注的图片放父笔记的 assets 文件夹，引用在批注块里
- **不再产生 `assets/images` 标签**：图片不再堆在全局目录，Obsidian 不会误识别

### 目录结构对比

**改之前**（图文分离）：
```
inBox/
├── 日记/
│   └── 生活/
│       └── 今天.md          ← 只有文字
└── assets/
    └── images/
        ├── img-001.jpg      ← 图片在这
        └── img-002.jpg
```

**改之后**（图文在一起）：
```
inBox/
└── 日记/
    └── 生活/
        ├── 今天.md           ← 正文里有 ![](今天-assets/img-001.jpg)
        └── 今天-assets/
            ├── img-001.jpg
            └── img-002.jpg
```

## 更新二：无标题笔记用正文前几个字当标题

### 问题
之前无标题笔记用日期当文件名，如 `2026-04-11 14.30.22.md`，看不出内容是啥。

### 改造
- **无标题笔记**：取正文前 20 个字当标题（文件名）
- **清洗规则**：去掉 `#标签`、markdown 语法符号（`#`、`*`、`>`、`-`、`![]()`、`[[]]` 等）、空白字符
- **有标题笔记**：完全不动，用原标题
- **纯图片/纯标签笔记**（清洗后正文为空）：回退到日期格式
- **日期不会丢**：日期保留在 frontmatter 的 `created` 字段里

### 效果示例

| 笔记内容 | 改之前的文件名 | 改之后的文件名 |
|---------|--------------|--------------|
| 有标题"今日记录" | `今日记录.md` | `今日记录.md`（不变） |
| 无标题，正文"今天去公园散步..." | `2026-04-11 14.30.22.md` | `今天去公园散步.md` |
| 无标题，正文"#日记 今天开心" | `2026-04-11 14.30.22.md` | `今天开心.md` |
| 无标题，纯图片无文字 | `2026-04-11 14.30.22.md` | `2026-04-11 14.30.22.md`（回退日期） |

## 改动的文件

- `src/sync/note-parser.ts` — `getLocalPath` 不再写死 `assets/images/`，localPath 改为只存文件名
- `src/sync/asset-handler.ts` — `handleAssets` 接收笔记 filePath 参数，资源下载到笔记同目录的 `笔记名-assets/` 子文件夹
- `src/sync/markdown-writer.ts` —
  - `getDisplayTitle`：无标题时取正文前 20 字（清洗后），空则回退日期
  - `generateMarkdown`：正文后追加资源引用区块（图片/视频/音频/附件）
  - `buildAnnotationBlock`：批注块里追加批注的资源引用
  - 新增 `extractContentPreview`、`buildAssetBlock`、`collectAssetLines`、`extractAssetFileName` 方法
- `src/sync/sync-manager.ts` — 调用 `handleAssets` 时传入笔记 filePath

## 升级建议

1. 编译生成新的 `main.js`
2. 把 `main.js` 和 `manifest.json` 复制到 vault 的插件目录
3. 建议先清空旧的 `inBox/assets/` 目录（里面是旧的全局图片），然后重新同步一次，图片会按新规则下载到各笔记旁边
4. 无标题笔记的文件名会在下次同步时自动更新为正文前几字

## 兼容性

- 有标题的笔记完全不受影响
- 旧的 `inBox/assets/` 目录里的图片不会被自动删除，需手动清理
- 配置项无变化，无需重新配置
