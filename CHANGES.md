# obsidian-inbox-sync 改造说明

针对原插件的两个不足做了改造：

## 问题一：批注单独成文件，与原文割裂

**原状**：每条批注是独立的 `.md` 文件，平铺在 `inBox/` 目录。父笔记末尾用 `![[批注文件名]]` 嵌入引用，但批注内容和原文没法对照着看。

**改造**：新增 `inlineAnnotations` 配置项（默认开启）。开启后：
- 批注不再生成独立文件
- 批注内容直接拼到父笔记末尾的「## 批注」区块
- 每条批注带标题、时间戳、正文
- 父笔记成为一份完整的、可对照阅读的文档

**效果示例**：
```markdown
---
title: 今日记录
inbox_id: note-abc123
tags:
  - 日记/生活
---

今天天气不错 #心情/开心

---

## 批注

### 补充
> _2026-04-11 14:30_

后来下雨了，带了伞。

### 回顾
> _2026-04-12 09:15_

那天其实没下雨，白带了。
```

关闭 `inlineAnnotations` 则回到旧的「独立文件 + 嵌入引用」模式。

## 问题二：笔记平铺，没有按标签聚合

**原状**：所有笔记平铺在 `inBox/` 一个目录里，标签只写进 frontmatter，没有任何目录组织。

**改造**：新增 `organizeByTag` 配置项（默认开启）。开启后：
- 按主标签（笔记的第一个标签）建子文件夹归档
- 支持嵌套标签：`#日记/生活` → `inBox/日记/生活/note.md`
- 无标签笔记留在根目录 `inBox/note.md`
- 其余标签仍保留在 frontmatter 的 `tags` 字段里

**目录结构示例**：
```
inBox/
├── 日记/
│   └── 生活/
│       ├── 2026-04-10.md
│       └── 2026-04-11.md
├── 技术/
│   └── 前端/
│       └── React学习笔记.md
└── 无标题-20260412.md   ← 无标签笔记留在根目录
```

可选配置 `tagFolderRoot`：如果不想标签目录直接建在 `inBox/` 下，可以指定一个父目录名，比如设为 `tags`，则路径变成 `inBox/tags/日记/生活/note.md`。

## 新增配置项

在 Obsidian 设置 → inBox 同步设置 → 笔记组织方式 中可配置：

| 配置项 | 默认值 | 说明 |
|--------|--------|------|
| 按标签分文件夹 | 开启 | 按主标签建子文件夹归档笔记 |
| 标签目录根 | 空 | 标签子文件夹的父目录名，留空则直接在存储路径下建 |
| 批注内联到父笔记 | 开启 | 批注内容拼到父笔记末尾，不单独生成文件 |

## 改动的文件

- `src/types/settings.ts` — 新增 3 个配置字段
- `src/sync/markdown-writer.ts` — 重写，支持标签分目录 + 批注内联
- `src/sync/sync-manager.ts` — 调整同步流程为两阶段（先解析收集批注，再写入）
- `src/ui/settings-tab.ts` — 新增配置 UI

## 安装

构建后把 `main.js` 和 `manifest.json` 复制到你 vault 的插件目录：
```
<vault>/.obsidian/plugins/inbox-sync/
```

然后在 Obsidian 设置中重新启用插件即可。新配置项默认开启，无需额外操作。

## 注意

- 首次启用新逻辑后，旧的平铺笔记不会自动迁移，建议先清空 `inBox/` 目录再重新同步一次
- 如果想保留旧的扁平结构，关闭「按标签分文件夹」和「批注内联到父笔记」即可回到原行为
